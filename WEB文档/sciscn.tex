\section{系统设计}

京沪高速铁路（Beijing-Shanghai High-speed Railway），简称京沪高铁，又名京沪客运专线，是一条连接北京市与上海市的高速铁路，是2016年修订的《中长期铁路网规划 》中“八纵八横”高速铁路主通道之一,
其所经区域面积是中国社会经济发展活跃的地区之一，也是中国客货运输较繁忙、增长潜力较大的客运专线。本项目基于Web前端开发技术，以HTML、CSS和JavaScript为
基本技术，综合应用EChart、SweetAlert等模板、技术完成对京沪高铁列车经停信息查询的前端设计与实现，主要功能包括路线图绘制、显示站点等。下面我将从项目设
计的系统目标、总体功能结构和主要技术路线三个方面进行介绍。
\subsection{系统目标}
本项目为京沪高铁列车经停信息查询的前端设计与实现，以京沪线和沿线经停高铁列车为主要呈现对象，要求项目设计与实现做到重点突出、导航明确、界面简洁、功能完
善和观赏性强五个方面。

从需求分析的过程来讲，本项目最核心的功能是完成京沪线及其沿线站点经停高铁列车信息的查询，因此，在进行前端设计的过程中，应重点突出这一功能模块，从视觉感官
上来讲，该功能模块在页面上的占比，决定着整个页面的功能导向。

此外，京沪铁路不仅在我国铁路网中发挥着不可替代的作用，其建设与营运中所涉及的技术也是国内乃至全球领先的技术，彰显了我国高铁技术的先进水平，所以除了之前提
到的的主要功能，本项目还旨在发挥京沪线和高铁技术的宣传作用这一角色。

当然，为保证系统的完整性，本项目也实现了注册与登录界面，单一的注册/登录界面显得枯燥乏味且不能体现系统的功能，脱离了系统的完整性。为了避免这一问题，我们浏
览和查阅了一些经典的注册/登录界面的案例，将注册/登录功能模块化，增加功能相关的动态元素，赋予静态页面以动的美感，同时不脱离系统完整性。

总而言之，本项目主要有注册/登录界面、主界面和基本基本介绍界面三个界面组成，本别承担着用户注册登录、高铁信息查询和基本信息介绍三个功能。在具体实现的过程中，通过
卡片布局、分栏、侧栏导航等设计方式突出系统的功能，便于用户对系统功能的了解、掌握和使用。

\subsection{系统总体功能}
系统的页面及功能组成如图\ref{fig1}所示.主要包括注册/登录界面、主界面和基本介绍界面，下面我将对三个界面及其功能进行介绍。
\subsubsection{注册/登录界面}
登录注册界面主要由纯CSS实现的高铁主题动画和注册登录功能模块组成。左侧是由纯CSS实现的高铁动画，契合了系统京沪高铁列车查询功能的主题，将系统功能形象直观地呈现
给用户，使得系统主题鲜明，功能相关，且动画的形式将静态页面赋予动的美感，不至于使得用户在注册登录过程中感到枯燥乏味。

右侧则是以卡片布局承载的注册/登录模块。这一模块以卡片布局的方式呈现，将页面的主要功能模块凸显出来，使得用户在使用系统时既能注意到主题动画，又不至于感到功能不
突出，从而体现了系统的重点突出和观赏性强的特点。

\subsubsection{主界面}
主界面主要有顶部、中部和底部三部分组成，其中顶部主要体现了系统的开发和所属单位，底部表明了作者信息和系统声明，而中部则是主界面的功能模块――京沪高铁列车信息
查询模块。
该功能模块以卡片布局和部分中国地图的形式为承载，将京沪线沿线省份突出显示，并将沿线各站标出，直观明了地将京沪线及其站点信息呈现给用户，而将列车信息隐藏与站点
中，突出了系统重点突出、功能完善、界面简洁和观赏性强的特点。
此外，页面顶部的页面导航将系统的页面进行串联，体现了系统功能晚上和导航明确的特点。
\subsubsection{基本介绍界面}
基本介绍页面同样分为顶部、中部和底部三部分，其中顶部主要体现了系统的开发和所属单位，底部表明了作者信息和系统声明，而中部则是基本介绍面的功能模块――京沪高铁列车信息
介绍模块。

该模块利用Div格局嵌套，将一个矩形进行区域切分，主要包括轮播图和文字介绍两部分。其中轮播图通过图片和循环滚动的方式，减少纯文字的枯燥，同时以直观的形式将京沪铁路的基本
信息传递给用户，而文字部分蕴含的信息和数据更加详实、准确，二者相辅相成、相得益彰。而大量的文字内容也不易于用户阅读，因此，我们通过添加侧边导航栏的方式提高阅读的便利
性，体现了系统易于导航和格局明确的特点。

\subsection{系统技术路线}
本项目的开发从软件工程生命周期的角度来看，分为需求分析、总体设计、详细设计、代码实现和系统测试五个部分，系统的技术路线如图\ref{fig3}所示.

\subsubsection{系统开发生命周期}
正如之前所提到的，在需求分析阶段，我们明确项目目的是前端设计与实现，呈现主题是京沪线及沿线车站高铁列车经停信息，整个项目以实现系统的完整性、美观性、功能性和明确性为
基本要求。

根据软件工程的分层思想，在总体设计阶段，对以上功能进行页面的划分和功能的扩展，根据功能划分为注册/登录界面、主界面和基本介绍界面，分别承担注册/登录功能、列车站点查询功能和基本信息
介绍功能，通过跳转、页面导航的方式降低功能之间的耦合性，增强每个功能模块的内聚性。同样，在每个页面中，通过Div分栏对功能进行分层，进一步降低细化功能间的耦合性。

在详细设计阶段，处于软件服务于用户的思想，将项目进行功能细化和界面美化，力求做到功能完善和浑然一体，界面简洁明确和具有较强的观赏性。以主界面为例，具体表现为突出展示
京沪铁路及其沿线省份，是用户对线路及站点信息一目了然，且通过自定义Alert对话框，优化展示界面和信息展示方式，给人以美的享受。

在代码实现阶段，采用软件工程演化过程模型，先进型原型开发，在完成原型的基础上不断迭代，知道实现初期设计要求。

在测试阶段，在完成单元测试的基础上进行集成测试和系统测试。

\subsubsection{系统开发主要技术}

在项目的实际开发中，主要采用了HTML、CSS、JavaScript等前端开发技术，而线路图的绘制和列车时刻的显示主要采用ECharts和SweetAlert.

HTML(Hypertext Markup Language)是超文本标记语言。它是一种标记语言，而不是编程语言。网页的内容包括标题、段落、表格、表单等。级联样式表(Cascading Style Sheet, CSS)，也称为层叠样式表。在设计Web网页时采用CSS技术，可以有效地对页面的布局、字体、颜色和其他效果实现更加精确的控制。采用CSS技术是为了解决网页内容与表现分离的问题。CSS
语言是一种标记语言，不需要编译，属于浏览器解释型语言，可以直接由浏览器解释执行。在HTML的基础上，使用JavaScript可以开发交互式Web页面。JavaScript的出现使得网页和用户之间
实现了一种实时性的、动态的、交互性的关系，是网页包含更多活跃元素和冯家精彩的内容。它是一种基于对象和事件驱动并具有相对安全性的客户端脚本语言。

此外，采用ECharts和jQuery进行线路和站点绘制。ECharts提供了丰富的可视化类型:如常规的折线图、柱状图，用于统计的盒形图，用于地理数据可视化的地图、热力图、线图，用于关系数据可视化的关系图等，
并且支持图与图之间的混搭。通过增量渲染技术，配合各种细致的优化，ECharts 能够展现千万级的数据量。而且还针对移动端交互做了细致的优化，例如移动端小屏上适于用手指在坐标系中进行缩放、平移。 PC 端也可以用鼠标在图中进行缩放（用鼠标滚轮）、平移等。

而站点高铁列车时刻的显示则借助了SweetAlert。SweetAlert 是一款不需要 jQuery 支持的原生 js 提示框,风格类似 bootstrap。SweetAlert允许通过HTML、CSS和JavaScript自定义，支持设置提示框标题、提示类型、内容展示图片、确认取消按钮文本、点击后回调函数等。

在实现静态页面的基础上，进一步利用CSS3的keyframes等属性，添加动态动画，将静态页面赋予灵动。

\subsubsection{系统开发设计模式}
在系统开发的设计模式上，借助组合模式的设计思想，将系统功能模块组合成树形结构（如图\ref{fig1}所示）以表示“整体-部分”的层次结构关系。组合是的对每个功能模块和符合功能模块的使用具有一致性。具体
表现为页面中能的划分和功Div分块的使用，既使得系统页面浑然一体，也保证了每个功能模块的独立性和可复用性。

 \par
 \setlength{\parindent}{2em}

\section{学习心得}
初接触Web前端开发技术，学到了关于URL和计算机网络的相关协议，仿佛给我打开了一扇计算机世界新的大门。随着学习的深入，慢慢接触了HTML的语言规范，也学会了CSS对HTML页面的渲染和通过定义JavaScript函数与用户进行交互实现更多的功能。正如之前所说，Web前端开发不是一门独立的学科，在我学习的过程中，我认为他与计算机网络，爬虫，用户界面设计与评价等课程的知识体系相互交织着。当我把一个含有文件下载的超链接部署在服务器的过程中，我学会了用HTML定义页面内容，用CSS对页面进行渲染和美化，用JavaScript实现其他编程语言也能实现的逻辑功能，用计算机网络的HTTP协议和TCP协议等进行数据传输……不仅如此，在学习了Web前端开发技术后，我会对日常中生活中遇到的设计良好的网页进行源码爬取和学习，也会根据网页返回的状态码进行排错和调试。
\par
\setlength{\parindent}{2em}当然，课堂学习的知识是有限的，Web前端设计知识的学习不能止步于此，在大项目的开发过程中，将软件工程的思想运用于项目开发中不仅使我对项目的开发驾轻就熟，而且也发现了相关知识体系的不足与漏洞。例如，大量的HTML文件和CSS文件以及JavaScript文件如何进行结构管理，整个项目开发是否有更加通用、敏捷的开发框架？这些问题在我进行项目开发的过程中得到了答案。Vue.js是很多生产场景中应用的开发框架，由于时间和学习成本的问题，很遗憾没有运用在本次项目中。但是，除此以外，还有很多很好的开源项目，如本次用到的ECharts、SweetAlert等，都值得我进一步去学习。

\begin{flushright}
\textbf{唐麒}
\end{flushright}

本学期初次接触到了web前端开发，每节课跟着老师一起读课本，确实收获很大很多知识点更有印象，能够弥补我之前上课很少看课本的缺点，每节课相应的小练习都很有成就感。本学期利用前端项目开发完成了两个大作业，一个是数据库实践中的合同管理系统，我负责前端开发部分，一个是我们的京沪高铁列车经停信息前端开发，html、css、div、JavaScript的使用，如何用css将页面渲染的更加美观，在开发过程中，我一直把用户使用友好放在重要位置，一个让用户看起来舒服的页面，操作方便简介明确，可以用下拉框代替用户输入等。但在两个大项目开发中能够明显感觉到之前自己所用知识的不足需要学习新的知识，例如vue框架、bootstrap、解决跨域问题等知识的学习使用。在这次小组作业的开发中我们利用了新的echart实现全国地图的绘制、动态路线绘制、星标站点等，这些都是新的尝试也会经历很多次失败的尝试。在开发过程中一个小小的点可能需要我们讨论很久，比如京沪高铁线路颜色和附近省份的颜色设置，看似很小的问题，但要实现用户友好我们进行了很多试验，前端开发对我个人而言比较有成就感，也清醒的认识到前端开发绝非易事需要前期的原型设计在实现过程中改进最初的想法，有些设计想起来很容易但在实际开发中会遇到各种问题，需要更细致的或者新的知识来帮助我们实现。总之，16周的web前端技术开发我学到不少之后也还要进行更深入的学习。
\begin{flushright}
\textbf{刘嘉欣}
\end{flushright}

\par
\setlength{\parindent}{2em}这学期选了web前端开发设计，有很多感悟。首先，我对web前端开发设计有浓厚的兴趣，web前端开发技术不只局限于网页设计，不再只是承载单一的文字和图片，软件化的交互形式为用户提供了更好的使用体验，同时它是一门概念性较强的交互式设计,艺术性较强的视觉设计。其次，通过老师基于仔细研读课本内容的上课方式和配合教学内容的作业，我在web前端开发过程中也有了一些学习web的心得。
\begin{itemize}
\item [(1)]一定要重视基础
\par
首先建立第一条技能线，前端由HTML,CSS,JavaScript组成；然后将其细化为第二条线，HTML是由HTML4和HTML5组成；第三条线，CSS由CSS2和CSS3组成；第四条线，JavaScript由es5和es6构成。
接下来是框架方面的延伸：CSS方向有less，HTML方向有bootstrap，js方向有jquery库
再往下延伸一层：HTML,CSS方向，有前端构建工具；js除了jq库之外，还有更全面的前端框架，react,angular,vue。前端知识点细小繁多，很多同学可能会摸不到头脑，不知道从哪里下手，其实掌握好基础而不是盲目的去贪多才是学习web的基础。
\item [(2)]注重实践
\par
在老师12次作业中，我深刻体会到设计一个网页的不易。首先要明确网页或网站的主题，围绕主题收集资料。其次，要考虑每一个布局，即使是一个很小的功能也要注重用户的友好性。最后，细化网页的过程要将我们所学的知识综合在一起，不断改进，才能有一个较好的结果。Web知识点细小很杂，只有在实践中不断运用，才能真正掌握。
\item [(3)]不断改进，综合运用
\par
前端不简单的地方，并不在于知识点的掌握，技术的实现，而在于寻找更优解。即使是最简单的页面布局，也存在层出不穷的优秀解决方案。这在最后的大作业中我深有体会。京沪铁路线路图，用传统的html语言也可以达成，但在美观程度和复杂程度上都不理想，于是我们小组学习了echart知识，又将它与所学的html,CSS,js知识结合起来，最终才完成了比较满意的京沪铁路查询网站。
\par
最后，关于张大林老师课堂方式，我认为回归课本是一种非常好的教学方式，可以让同学们对基础知识有更深刻的了解，根据课堂内容布置作业也很有利于大家掌握所学知识。希望大林老师的web课越来越受同学们喜欢！
\end{itemize}

\textbf{张雨桐}

在本节课中，我学习到了很多，对javascript，css以及html也有了更加深层次的了解，也对web前端技术的结构有了更加完整的认识。我很喜欢老师的上课方式，带着大家看书却是弥补了我们现在很少看书的问题。我觉得老师应该花更少的时间在和大家讨论每一段代码的作用上，应该为我们拓展更多的知识，这样上课总觉得浪费了时间。而且这样上课总觉得对知识的掌握还不够，虽然每一次的作业都是在上课后再把代码敲出来进行实践，但还是总觉得好多知识都记不住，实际的操作还不够。这节课上学习的东西在未来的学习中一定还会用到很多，很多的项目都会使用网页，我认为设计一个很漂亮的网页会让人的心情舒畅，是一件很开心的事情，所以我还是很喜欢做网页的。这学期我们在数据库实践课的项目上就使用了一些网页的技术，在答辩的时候，我就觉得一个漂亮的前端网页会使人眼前一亮，结合强大的后台算法技术一定会是一个很成功的项目，会是大学生活中很自豪的一个项目。最后，建议老师继续督促大家认真看书，获取知识，也希望以后的教学中会督促大家做更多的实践，加强大家对代码的记忆使大家可以更好的设计出自己的漂亮的网页。这学期在大林老师的带领下学习web知识真的很开心，学到的东西也很丰富且实用，谢谢老师，一切祝好。

\end{aligned}
\right.
